LLDP Deployment

Automated LLDP Installation and Configuration for Linux Servers using Ansible

What is LLDP?

LLDP (Link Layer Discovery Protocol) is an industry-standard network protocol that enables network devices to advertise their identity, capabilities, and neighbors on a local area network. It helps network administrators:

  • Discover network topology automatically
  • Identify which switch port a server is connected to
  • View neighboring devices and their capabilities
  • Troubleshoot network connectivity issues
  • Document network infrastructure

What Does This Utility Do?

This utility provides Ansible playbooks and automation tools to deploy and configure the lldpd daemon across multiple Linux servers simultaneously. It simplifies the process of:

Automated Installation

Installs and configures lldpd on Debian/Ubuntu and RedHat/CentOS systems with a single command.

Non-Root Access

Configures passwordless sudo access for non-root users to run LLDP commands securely.

Service Management

Ensures lldpd service starts on boot and manages systemd configuration automatically.

Neighbor Discovery

Provides simple commands to view LLDP neighbors and network topology across all servers.

Compatibility

Works with systems running as root (Proxmox) and regular users with sudo access.

Troubleshooting

Includes diagnostic tools to detect and fix common issues like requiretty and sudo configuration.

Quick Start

Prerequisites: Ansible 2.9 or higher, SSH access to target servers, and sudo privileges.

Method 1: Interactive Quickstart (Recommended)

The easiest way to get started is using the interactive quickstart script:

./quickstart.sh

This will guide you through the entire setup process, automatically detecting and fixing common issues.

Method 2: Manual Installation

# 1. Clone the repository git clone https://github.com/soothill/lldp-deploy.git cd lldp-deploy # 2. Create inventory file cp example_inventory.ini inventory.ini nano inventory.ini # Add your servers # 3. Test connectivity make test-connection # 4. Deploy LLDP make install # 5. View neighbors make neighbors

Installation Steps

1Create Your Inventory File

Copy the example inventory and add your servers:

cp example_inventory.ini inventory.ini

Example inventory.ini:

[lldp_servers] server1.example.com ansible_host=192.168.1.10 ansible_user=admin server2.example.com ansible_host=192.168.1.11 ansible_user=ubuntu proxmox.example.com ansible_host=192.168.1.20 ansible_user=root ansible_become=no [lldp_servers:vars] ansible_user=ubuntu

2Test Connectivity

Verify Ansible can connect to your servers:

make test-connection

3Deploy LLDP

Install and configure LLDP on all servers:

# Basic installation make install # Or with password prompt if needed make install ANSIBLE_OPTS='--ask-become-pass' # Advanced installation with custom config make install-advanced
What happens during installation:
  • Installs lldpd package on target systems
  • Creates lldpd group for socket access
  • Adds current user to lldpd group
  • Configures systemd service and socket permissions
  • Sets up passwordless sudo for LLDP commands
  • Enables and starts the lldpd service

4Verify Installation

# Check service status make verify # View LLDP neighbors make neighbors # Show chassis information make info

Usage Examples

Deploy to Multiple Servers

# Deploy to all servers in inventory make install # Check what would change without making changes make check # Deploy with verbose output for troubleshooting make install ANSIBLE_OPTS='-vv'

Deploy to Specific Servers

Troubleshooting

"Missing sudo password" Error

This is the most common issue during deployment. It can be caused by several factors.

Diagnosis:

ansible-playbook -i inventory.ini check_sudo.yml

Common Fixes:

1. requiretty Issue (Most Common)

Symptoms: "sudo: a password is required" even though passwordless sudo works via SSH

# Automatic fix ansible-playbook -i inventory.ini fix_requiretty.yml --ask-become-pass

2. Passwordless Sudo Not Configured

ansible-playbook -i inventory.ini setup_passwordless_sudo.yml --ask-become-pass

3. Need to Provide Password

If your user requires a sudo password, add the password prompt flag:

make install ANSIBLE_OPTS='--ask-become-pass'

Non-Root Users Can't Run LLDP Commands

After installation, non-root users need to activate their lldpd group membership:

# Either log out and log back in, or run: newgrp lldpd # Then run LLDP commands with sudo (no password required): sudo lldpcli show neighbors
Why sudo? Using sudoers with NOPASSWD is more secure than changing binary permissions and survives package updates. Users in the lldpd group can run LLDP commands without a password.

Service Not Starting

# View detailed logs make logs # Or manually on the server journalctl -u lldpd -n 50 # Check service status systemctl status lldpd

No Neighbors Detected

  • Ensure network switch supports LLDP and it's enabled on switch ports
  • Verify network connectivity
  • Wait 30-60 seconds for neighbor discovery to complete
  • Check firewall rules aren't blocking LLDP packets

Advanced Features

Custom Configuration

The advanced playbook supports custom LLDP configuration via templates:

# Deploy with custom config make install-custom # Edit the template at templates/lldpd.conf.j2 # Configure system description, platform, SNMP, etc.

Configuration Variables

Customize behavior in your inventory file:

[lldp_servers:vars] lldp_package=lldpd lldp_service=lldpd lldp_system_description="My Server" lldp_system_platform="Linux" lldp_enable_snmp=no lldp_mgmt_pattern="!dummy*,!veth*"

Parallel Deployment

Speed up deployment across many servers:

# Deploy to 10 servers in parallel make install ANSIBLE_OPTS='-f 10'
